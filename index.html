Key Insight: Why These Differences Matter
1
Arrays vs Linked Lists: Arrays win on access O(1) because memory is contiguous — the CPU computes address = base + index × size. Linked lists win on front insertion O(1) because no shifting is needed.
2
Singly vs Doubly: Adding a prev pointer trades space for time. Back-deletion drops from O(n) to O(1) — the biggest win.
3
Recursion space: Every recursive call uses a stack frame. A loop uses O(1) space; equivalent recursion uses O(n). Tail-call optimization can fix this — but Java/Python don't do it by default.
Core Concepts
Step-by-step walkthroughs of key operations
Loop → Recursion Conversion
1
Identify the loop variable. Every for/while loop has a variable that changes each iteration (e.g., i, current). This becomes your recursion parameter.
2
Base case = loop termination. When does the loop stop? i >= n or current == null. That exact condition becomes your if (base_case) return.
3
Recursive call = loop body + increment. Do the loop body's work, then call yourself with the updated parameter: f(i+1) or f(current.next).
4
Example: for(i=0;i<n;i++) print(arr[i]) becomes void print(arr, i, n) { if(i==n) return; print(arr[i]); print(arr, i+1, n); }
Singly Linked List: Insert at Front
1
Create a new node: Node newNode = new Node(data)
2
Point new node's next to current head: newNode.next = head
3
Update head to new node: head = newNode
4
Why O(1)? We only touch 2 pointers — no traversal needed.
Singly Linked List: Delete from Back
1
Edge case: If list has 0 or 1 node, handle separately.
2
Traverse until current.next.next == null — stop at the second-to-last node.
3
Set current.next = null — this severs the last node.
4
Why O(n)? We have no prev pointer, so we must walk the whole list to find the predecessor.
Doubly Linked List: Delete from Back
1
Access tail.prev — the second-to-last node. Instantly available!
2
Set tail.prev.next = null
3
Update tail = tail.prev
4
Why O(1)? The prev pointer gives us direct access to the predecessor — no traversal.
Circular Linked List: Key Quirks
1
No null terminator: The last node's next points back to head. Loop detection uses current != head as the stop condition.
2
Insert at front: You must also update tail.next = newHead. Forgetting this breaks the circle.
3
Infinite loop risk: Always track when you've returned to head, or your traversal loops forever.
4
Use case: Round-robin scheduling, music playlists, any cyclic rotation problem.
Recursion: The Call Stack Analogy
1
Think of recursion as a stack of sticky notes. Each call writes its local variables on a note and stacks it on top.
2
The base case is the point where you stop adding notes and start peeling them off.
3
When you peel off a note (return), the previous note resumes — this is how return values propagate back up.
4
Stack overflow = too many notes. Missing or wrong base case keeps adding notes until memory runs out.
